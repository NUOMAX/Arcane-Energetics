Обзор проекта

Философия мода

Arcane Energetics — это не просто мод, а целая философия инженерии в мире Hytale, где техника, магия и природа существуют в симбиозе.

Технологии

Реалистичные механизмы с физикой, износом и модульной сборкой. От примитивных шестерней до сложных автоматизированных систем.

Магия

Не альтернатива технологиям, а их дополнение. Магические кристаллы, рунические гравировки, природные духи, взаимодействующие с механизмами.

Природа

Не просто ресурс, а активный участник. Биомы влияют на механизмы, животные взаимодействуют с ними, растения могут быть компонентами.

Люди (игрок)

Инженер-первооткрыватель, который не просто крафтит, а проектирует, экспериментирует и находит баланс между прогрессом и гармонией.

Tier 1: Примитивная инженерия

Обзор Tier 1

Игрок начинает как робот, не помнящий своего прошлого, в разрушенной лаборатории. Он должен заново открыть основы механики, используя примитивные материалы: дерево, камень, верёвки, кожу.

Механики и системы

Система точности и калибровки

Каждый механизм имеет коэффициент точности (0-100%), влияющий на его эффективность, шум и износ.

Высокая точность (80-100%)

  • +20% эффективности
  • -30% износа
  • Тихая работа
  • Визуальные эффекты точности

Средняя точность (50-80%)

  • Базовая эффективность
  • Нормальный износ
  • Умеренный шум
  • Привлекает внимание мобов

Низкая точность (0-50%)

  • -30% эффективности
  • +50% износа
  • Громкий шум
  • Высокий риск поломки

Сетевые эффекты и резонанс

Когда несколько механизмов работают в унисон и синхронно, они создают резонансное поле с дополнительными эффектами.

Количество механизмов Бонус эффективности Дополнительные эффекты Визуальные эффекты
3+ +10% Синхронный звук Лёгкая вибрация
6+ +20% -15% износа, гармоничный гул Ритмичное свечение
10+ +30% Резонансное поле: отталкивает мобов, ускоряет рост растений Видимые волны энергии, звуковая аура

Инструменты для работы с механизмами

Угольник

Проверка углов 90°. Повышает точность сборки на +5% за проверенный угол.

Регулировочные клинья

Тонкая настройка положения деталей. +1-3% точности за клин.

Уровень с водой

Проверка горизонта. Критично для водяных колёс и прессов.

Примитивный термометр

Измерение температуры. Важно для металлургии (намёк на Tier 2).

Биомы Hytale и их влияние

Лесной биом

Плюсы: Дерево в изобилии, смола с хвойных

Минусы: Корни повреждают подземные механизмы, животные атакуют деревянные конструкции

Особенности: +10% эффективности водяных колёс

Пустынный биом

Плюсы: Солнечная энергия (зеркала-концентраторы)

Минусы: Песок забивает механизмы, перегрев деталей днём

Особенности: Кожа быстро высыхает и трескается

Снежный биом

Плюсы: Лёд для улучшенных подшипников

Минусы: Обледенение механизмов, металл становится хрупким

Особенности: Водяные колёса замерзают, нужен антифриз

Вулканический биом

Плюсы: Высокая температура для плавки, базальт

Минусы: Пеплопад засоряет механизмы, кислотные дожди

Особенности: Дерево может самовоспламеняться

// Пример класса механической энергии public class MechanicalEnergySystem { private static final double BASE_EFFICIENCY = 0.85; private static final double DISTANCE_LOSS_PER_BLOCK = 0.02; public static double calculateMEOutput(EnergySource source, int distance) { double baseOutput = source.getBaseOutput(); double efficiency = BASE_EFFICIENCY - (distance * DISTANCE_LOSS_PER_BLOCK); // Учёт погодных условий if (source instanceof WaterWheel) { efficiency *= WeatherSystem.getWaterFlowMultiplier(); } return Math.max(0, baseOutput * efficiency); } public static boolean checkOverload(MechanicalPart part, double load) { double maxLoad = part.getMaterial().getMaxLoad(); double currentWear = part.getWearLevel(); // Износ снижает максимальную нагрузку double effectiveMaxLoad = maxLoad * (1 - currentWear * 0.5); return load > effectiveMaxLoad; } } // Пример класса детали public class GearPart { private Material frameMaterial; private Material teethMaterial; private Coating coating; private double wearLevel = 0.0; private double precision = 1.0; public double calculateEfficiency() { double baseEfficiency = 0.9; double materialBonus = frameMaterial.getEfficiencyBonus() + teethMaterial.getEfficiencyBonus(); double coatingBonus = coating.getEfficiencyBonus(); double wearPenalty = wearLevel * 0.3; double precisionBonus = (precision - 0.5) * 0.2; return Math.max(0.1, baseEfficiency + materialBonus + coatingBonus - wearPenalty + precisionBonus); } }

Принципы работы команды

1. Уважение к мнению каждого

Каждый член команды имеет право голоса. Решения принимаются коллективно.

2. Мод важнее разногласий

Личные конфликты не должны влиять на разработку. Мод — наш общий приоритет.

3. Качество важнее скорости

"Поспешишь — людей насмешишь". Мы не гонимся за сроками, а делаем качественно.

4. Реальная жизнь важнее

Сначала решаем проблемы в реальной жизни, потом возвращаемся к разработке.

5. Наслаждайтесь разработкой!

Создание мода должно приносить удовольствие, а не быть рутиной.
